This submission implements a LangGraph-style multi-memory agent without requiring an LLM API key.

## Files

- `memory_agent.py`: LangGraph `StateGraph`, memory backends, router, prompt injection, context trimming, optional OpenAI calls.
- `run_benchmark.py`: runs 10 multi-turn benchmark conversations and writes `BENCHMARK.md`.
- `test_memory_agent.py`: unit tests for conflict handling.
- `BENCHMARK.md`: benchmark report with no-memory vs with-memory comparison.
- `data/`: JSON memory files generated by the benchmark.

## Run

```bash
python run_benchmark.py
python -m unittest -v
```

To enable OpenAI-based extraction/generation:

```bash
python run_benchmark.py --llm
```

The model defaults to `gpt-4o-mini-2024-07-18` via `OPENAI_MODEL`.

## Memory Stack

- Short-term: in-memory conversation buffer.
- Long-term profile: JSON key-value store.
- Episodic: JSON event log.
- Semantic: Chroma retrieval over seeded document chunks with keyword fallback.
- Context budget: `tiktoken` token counting with character-count fallback.
